**学习内容**：

**必读论文**：
1. **ReAct**（必读！）
   - 论文：https://arxiv.org/abs/2210.03629
   - 核心思想：Reasoning + Acting 交替进行
   - 算法原理：如何设计推理-行动循环

**学习任务**：
- [ ] 阅读 ReAct 论文，理解算法流程

### 🗓️ Week 3-4：Agent Memory 与上下文工程算法

**学习目标**：
- 掌握 Memory 机制的算法设计
- 理解上下文压缩算法
- 设计自己的 Memory 优化方案

#### Day 15-18：Memory 算法

**学习内容**：
- [ ] 短期记忆 vs 长期记忆
- [ ] 记忆重要性评分算法
- [ ] 记忆压缩与总结策略
- [ ] 记忆检索优化

**核心算法点**：
1. **重要性评分**：
   - 如何判断哪些信息应该存入 Memory？
   - 算法设计：语义相似度 + 任务相关性 + 时效性

2. **压缩算法**：
   - 如何在保留关键信息的同时压缩存储？
   - 算法：聚类 + 摘要 + 去重

3. **检索优化**：
   - 如何快速从海量记忆中检索相关信息？
   - 算法：向量检索 + 时间衰减 + 重要性加权

**学习任务**：
- [ ] 分析 Mem0、MemGPT 的 Memory 设计
- [ ] 设计一个分层记忆架构
- [ ] 实现一个简单的记忆压缩算法

**面试准备**：
- Q: 如何设计 Agent 的长期记忆机制？
- Q: 记忆压缩和检索的trade-off如何平衡？

**学习内容**：
- [ ] 上下文选择策略

**必读论文**：

   - 论文：https://arxiv.org/pdf/2510.26493

**核心算法**：

   - 逻辑依赖（任务步骤依赖）
   - 时效性（时间衰减）

2. **上下文压缩**：
   - 层级笔记压缩

**学习任务**：

#### Day 22-28：强化学习 + Agent

**学习内容**：
- [ ] RL 基础：MDP、Q-learning、Policy Gradient
- [ ] Agent + RL 的结合点
- [ ] 奖励函数设计
- [ ] 策略优化算法

**必读论文**：
1. **Tool Learning with RL**
   - 核心思想：用 RL 优化工具调用策略

2. **GRPO**（最新）
   - 核心思想：Group Relative Policy Optimization
   - 算法创新：相对偏好建模

**核心算法点**：
1. **奖励设计**：
   - 任务成功 vs 中间步骤
   - 稀疏奖励 vs 密集奖励

2. **策略优化**：
   - PPO vs DPO vs GRPO
   - 如何平衡探索和利用？

**学习任务**：
- [ ] 手撕 DPO 算法

## 💡 算法岗学习建议

### ✅ Do's（应该做的）

1. **深入理解原理**
   - 不只是会用，要知道为什么
   - 能推导公式，能讲清楚数学原理
   - 看论文要看到 Method 和 Experiments 部分

2. **严谨的实验**
   - 一定要做对比实验
   - 一定要做消融实验
   - 用数据说话，不能主观判断

3. **持续产出**
   - 写技术博客，加深理解
   - 开源代码，建立影响力
   - 尝试投论文，即使是 Workshop

4. **关注前沿**
   - 每周浏览 ArXiv
   - 关注顶会 Accept 论文
   - 复现经典算法

### ❌ Don'ts（不要做的）

1. **不要只调包**
   - LangChain/LlamaIndex 只是工具
   - 要理解底层原理
   - 必要时要能手撕实现

2. **不要忽视实验**
   - Demo 效果好 ≠ 算法有效
   - 一定要在标准数据集上验证
   - 一定要对比 baseline

3. **不要闭门造车**
   - 多看论文，站在巨人肩膀上
   - 多交流，参加学术会议/讨论
   - 多开源，获得社区反馈
